## 第一步：阿里云购买域名

### 1.1 注册阿里云账号

1. 打开 [aliyun.com](https://aliyun.com)
2. 点击右上角"免费注册"，使用你的邮箱（19507173502@163.com）注册

### 1.2 域名查询与购买

1. 登录后，在顶部搜索框输入你想要的域名，点击"查域名"
2. 推荐后缀和命名方案：

| 方案 | 域名示例 | 年费参考 | 推荐理由 |
|------|---------|---------|---------|
| **推荐：`.com`** | `xuelinezhishang.com` | ¥55-79/年 | 全球通用，最权威 |
| **推荐：`.art`** | `xuelinezhishang.art` | ¥65-89/年 | 文艺属性契合纪录片调性 |
| 备选：`.film` | `xuelinezhishang.film` | ¥199+/年 | 行业相关但较贵 |

## 第四步：GitHub Pages 开启与绑定自定义域名

### 4.1 开启 GitHub Pages

1. 进入你的仓库页面：`https://github.com/你的用户名/xueline-zhishang`
2. 点击顶部导航栏的 **Settings**（设置）
3. 在左侧菜单栏点击 **Pages**（位于 "Code and automation" 分组下）
4. 在 "Branch" 部分：
   - **Source**：选择 **"Deploy from a branch"**
   - **Branch**：选择 **main**，文件夹选 **/（root）**
   - 点击 **Save**
5. 等待约 1-2 分钟，页面顶部会出现绿色提示：
   ```
   Your site is live at https://你的用户名.github.io/xueline-zhishang/
   ```
   **此时可通过这个 `github.io` 临时地址访问网站。**

### 4.2 绑定自定义域名（请先完成第五步的 DNS 解析后再回来操作）

> 注意：DNS 解析和 GitHub 域名绑定需要配合完成。**推荐顺序：先在阿里云配好 DNS → 再回 GitHub 绑定域名。**

1. 在同一个 Pages 设置页面，找到 **"Custom domain"** 输入框
2. 输入你在阿里云购买的域名，例如：`xuelinezhishang.com`
3. 点击 **Save**
4. GitHub 会自动检测该域名的 DNS 记录是否已配置正确
5. 如果提示 "DNS check successful"，说明配置正确
6. **不要勾选 "Enforce HTTPS"**（等第五步完成后再开启）

## 第五步：阿里云 DNS 解析配置（CNAME 仅且唯一）

> 这是最关键的一步——告诉阿里云：当用户访问你的域名时，去 GitHub Pages 找网站内容。

### 5.1 进入云解析 DNS 控制台

1. 登录阿里云，进入 **域名控制台**（控制台首页 → 域名）
2. 找到你购买的域名，点击右侧 **"解析"**（或 "DNS设置"）
3. 进入 **"云解析 DNS"** 控制台

### 5.2 添加 CNAME 记录

1. 点击 **"添加记录"**
2. 填写以下参数：

   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **记录类型** | **CNAME** | 唯一需要的类型 |
   | **主机记录** | **@** | 表示根域名（如 xuelinezhishang.com） |
   | **记录值** | **你的用户名.github.io** | 替换"你的用户名"为你的 GitHub 用户名（全小写） |
   | **TTL** | **600**（10分钟）或 **自动** | 缓存时间，默认即可 |

   > 示例：如果你的 GitHub 用户名是 `MingxinXiang`，记录值填 `mingxinxiang.github.io`

3. 点击 **"确认"**

### 5.3 添加 www 子域名转发（可选）

如果需要 `www.xuelinezhishang.com` 也能访问：

1. 再次点击 **"添加记录"**
2. 填写：

   | 字段 | 填写内容 |
   |------|---------|
   | 记录类型 | **CNAME** |
   | 主机记录 | **www** |
   | 记录值 | **你的用户名.github.io** |
   | TTL | 600 |

### 5.4 重要提醒

- **不要添加 A 记录** —— GitHub Pages 的服务器 IP 会变，A 记录会导致网站突然打不开
- **不要使用国内 URL 转发服务** —— 不稳定且可能被拦截
- **DNS 生效时间**：新添加的记录通常在 **10 分钟到 2 小时内**全球生效

## 第六步：GitHub 强制开启 HTTPS

1. 回到你的 GitHub 仓库 → **Settings** → **Pages**
2. 在 "Custom domain" 区域下方，找到 **"Enforce HTTPS"**
3. **勾选 "Enforce HTTPS"**（只有域名 DNS 验证通过后，此选项才可点击）
4. 等待几分钟，GitHub 会自动为你的域名申请 Let's Encrypt SSL/TLS 证书
5. 证书申请完成后，所有访问都将自动加密（https:// 开头）
6. 验证：在浏览器中打开 `https://你的域名`，查看地址栏左侧是否有 **锁形图标**

## 第七步：解析生效验证与多设备访问测试

### 7.1 验证 DNS 是否生效

**方法一：命令行验证（推荐）**

打开命令提示符（Win+R → 输入 `cmd` → 回车），输入：

```bash
nslookup xuelinezhishang.com
```

如果返回结果中包含 `github.io` 相关地址，说明解析已生效。

**方法二：在线工具验证**

访问 [whatsmydns.net](https://www.whatsmydns.net/)，输入你的域名，选择 CNAME 查询，可查看全球各地区的 DNS 生效状态。

### 7.2 浏览器访问测试

**电脑端：**
- 在 Chrome/Edge 浏览器地址栏输入 `https://xuelinezhishang.com`
- 检查：页面是否完整加载、图片是否显示、导航是否正常、布局是否错位
- 按 F12 打开开发者工具 → 点击手机图标 → 选择不同设备尺寸测试响应式适配

**手机端：**
- 在手机浏览器中输入 `https://xuelinezhishang.com`
- 检查：触控是否灵敏、字体大小是否合适、菜单按钮能否展开

### 7.3 速度优化检测

- 使用 [PageSpeed Insights](https://pagespeed.web.dev/) 测试网站加载速度
- GitHub Pages 全球 CDN 加速，通常得分在 90 分以上

## 第八步：后续更新维护流程

### 8.1 修改文案

1. 在桌面 `雪线之上_官网` 文件夹中找到 `index.html`
2. 用 **记事本** 或 **VS Code** 打开
3. 直接找到对应文字进行修改（Ctrl+F 搜索关键词）
4. 修改完成后保存文件
5. 在仓库页面点击 **"Add file" → "Upload files"**，重新上传新的 `index.html`
6. 在 "Commit changes" 框中说明修改内容，点击提交
7. GitHub Pages 会在 1-3 分钟内自动更新

### 8.2 替换海报/图片

1. 准备新的图片（建议 PNG 格式，单图不超过 2MB）
2. 将新图片放入 `雪线之上_官网` 文件夹
3. 在 `index.html` 中找到对应的 `<img>` 标签或 CSS 背景，修改图片文件名
4. 同时上传 `index.html` + 新图片到 GitHub 仓库

### 8.3 新增图表

1. 导出新图表为 PNG 文件
2. 放入 `雪线之上_官网` 文件夹
3. 在 `index.html` 中找到 "创作逻辑可视化图表" 区域，参考现有 `chart-item` 结构添加新的图表卡片
4. 在 `<script>` 区域添加新的 `tryLoadChart()` 调用
5. 上传所有更新文件到 GitHub 仓库

### 8.4 快速更新清单

| 修改类型 | 需上传的文件 | Pages 更新耗时 |
|---------|------------|--------------|
| 仅改文字 | `index.html` | 1-3 分钟 |
| 替换图片 | `index.html` + 新图片 | 1-3 分钟 |
| 新增板块 | `index.html` | 1-3 分钟 |
| 新增图片/图表 | `index.html` + 新增的 .png 文件 | 1-3 分钟 |

### 8.5 本地直接预览

在更新上传之前，你可以随时双击桌面 `雪线之上_官网/index.html`，在浏览器中本地预览效果。所有内容在本地确认无误后再上传到 GitHub。
